# ssh-server-runner

A script to run an `ssh` + `dtach` server, designed to work seamlessly inside devcontainers.

It installs the missing apt dependencies needed to run `sshd`, bootstraps a few CLI tools, configures SSH access for the current user, and starts the SSH server on the chosen port.

If available, the script also forwards the latest `SSH_AUTH_SOCK` generated by VS Code, allowing you to use your local SSH keys inside the server.

## Quick Start

Run this one-liner inside your devcontainer terminal:

```bash
# Start the server on port 5001
curl -fsSL https://raw.githubusercontent.com/PabloZaiden/ssh-server-runner/main/ssh-server.sh | SSH_PORT=5001 bash
```

If VS Code doesn't automatically expose the port, you can forward it manually.

The script does not implement a `--stop` flag. To stop the server, terminate the `sshd` process that was started by the script.

## What the script installs and configures

`ssh-server.sh` does more than just start `sshd`. It installs software, writes configuration, and creates local state so later runs can reuse it.

### Apt packages

The script checks whether these Debian packages are already installed and only installs the ones that are missing:

- `openssh-server` for the SSH daemon itself
- `uuid-runtime` so it can generate a random password with `uuidgen`
- `dtach`

It also creates these runtime/configuration directories if they do not already exist:

- `/var/run/sshd`
- `/etc/ssh/sshd_config.d`

### Bootstrapped tools

The script also bootstraps these tools:

- `fresh` editor, installed by piping the upstream install script from `https://raw.githubusercontent.com/sinelaw/fresh/refs/heads/master/scripts/install.sh` into `sh`
- GitHub CLI (`gh`), but only if `gh` is not already available on `PATH`
- GitHub Copilot CLI, installed by piping `https://gh.io/copilot-install` into `bash`

If `gh` is missing and the default apt repositories do not provide it, the script installs the extra packages needed to add GitHub's official apt repository:

- `ca-certificates`
- `curl`
- `gnupg`

It then writes:

- `/etc/apt/keyrings/githubcli-archive-keyring.gpg`
- `/etc/apt/sources.list.d/github-cli.list`

before retrying the `gh` install.

### SSH authentication and generated files

On first run, the script generates a random password and stores it in `CRED_FILE`, which defaults to `./.sshcred`.

On later runs, it reuses the same credential file instead of rotating the password again. If the credential file already exists but is empty, the script exits with an error.

If the credential file lives inside the current git repository, the script adds its repo-relative path to `.git/info/exclude` so the password file is ignored locally without modifying `.gitignore`.

### SSH daemon configuration

The script configures SSH for the current user (preferring `SUDO_USER` when invoked through `sudo`) and ensures that user exists before setting the password.

It writes `/etc/ssh/sshd_config.d/99-local.conf` with:

- `Port ${SSH_PORT}` where `SSH_PORT` defaults to `5001`
- `PasswordAuthentication yes`
- `KbdInteractiveAuthentication yes`
- `UsePAM yes`
- `PermitRootLogin yes` when the current user is `root`, otherwise `PermitRootLogin no`

If `/etc/ssh/sshd_config` does not already include `/etc/ssh/sshd_config.d/*.conf`, the script appends that include line once.

### SSH agent forwarding persistence

If the environment already contains a usable `SSH_AUTH_SOCK` value, the script appends matching `export SSH_AUTH_SOCK="..."` lines to:

- `~/.profile`
- `~/.bashrc`
- `~/.zshenv`

Before doing that, it prefers the newest VS Code-generated socket matching `/tmp/vscode-ssh*.sock` when one is available.

### What happens every run vs only when needed

Some actions are conditional:

- apt packages are only installed when they are missing
- `gh` is only installed when it is not already present
- the GitHub CLI apt repository is only added if the normal apt install for `gh` fails
- the password file is only created on first run
- the `.git/info/exclude` entry is only appended when the credential file is inside the repository and the pattern is not already present

Other actions are attempted on each run:

- `fresh` is bootstrapped
- GitHub Copilot CLI is installed
- SSH config is written/refreshed
- `sshd` is started on the selected port

## Authentication

When the script finishes, it prints:

- the SSH user
- the generated or reused password
- the SSH port
- the effective `PermitRootLogin` value

Use that username, password, and port to connect to the server from your SSH client.
