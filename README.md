# ssh-server-runner

A script to run an `ssh` + `dtach` server, designed to work seamlessly inside devcontainers.

It will install `ssh` and `dtach` if not already available, generate a random password on first run, and reuse it on subsequent runs. The server will be accessible on the forwarded port.

If available, the script will also forward the latest `SSH_AUTH_SOCK` generated by VS Code, allowing you to use your local SSH keys inside the server.

## Quick Start

Run this one-liner inside your devcontainer terminal:

```bash
# Start the server on port 5001
curl -fsSL https://raw.githubusercontent.com/PabloZaiden/ssh-server-runner/main/ssh-server.sh | SSH_PORT=5001 bash
```

```bash
# Stop the server
curl -fsSL https://raw.githubusercontent.com/PabloZaiden/ssh-server-runner/main/ssh-server.sh | bash -s -- --stop
```

If VS Code doesn't automatically expose the port, you can forward it manually

## Authentication

On first run, the script will generate a random password for the `ssh` server and print it to the terminal. You can use this password to connect to the server.

On subsequent runs, the script will reuse the same password, which is stored in `./.sshcred`. If running in a git repository, the file will locally added to the exclude list to avoid committing it by accident.
